📋 Следующий шаг (Приоритет 2) – Фронтенд на React
Цель: создать клиентскую часть, которая будет взаимодействовать с уже готовым API.
Конкретные задачи:
Создать React-приложение

Использовать Vite (быстрее) или CRA

Папка shop_frontend/ в корне проекта

Настроить прокси / базовый URL для API

В vite.config.js прописать прокси на http://localhost:3000

Или создать .env с VITE_API_URL

Реализовать страницы

Страница логина (/login) – форма email + password, отправка на POST /auth/login, сохранение токена в localStorage (или httpOnly cookie – обсудим)

Страница регистрации (/register)

Главная страница / Каталог товаров (/) – список товаров с карточками (название, цена, бренд)

Страница деталей товара (/product/:id) – полная информация

Админ-панель (/admin) – доступна только для пользователей с ролью admin. Внутри:

Управление товарами (добавление, редактирование, удаление)

Управление категориями и брендами

Управление покупателями/поставщиками (из CRM)

Работа с токеном

Создать axios инстанс с интерсептором:

При каждом запросе добавлять заголовок Authorization: Bearer <token>

При получении 401 – перенаправлять на логин

Маршрутизация

Использовать react-router-dom

Приватные маршруты (например, /admin) проверяют роль пользователя (хранить роль в localStorage или декодировать JWT)

Стилизация (опционально на старте)

Можно использовать простой CSS или UI-библиотеку (Material UI, Ant Design, Tailwind) для ускорения

Проверка результата
Пользователь может зарегистрироваться, войти, просматривать товары.

Администратор (учётная запись с ролью admin) может создавать/редактировать/удалять товары, категории, бренды через интерфейс.

Пример запуска
bash
# В папке shop_frontend
npm install
npm run dev   # фронт на порту 5173 (Vite по умолчанию)
# Бэкенд уже запущен на 3000
📌 Примечания к текущему состоянию
База данных SQLite с синхронизацией – для разработки.

JWT секрет лежит в .env, не закоммичен (добавить .env в .gitignore).

Валидация DTO работает через class-validator.

Отношения между сущностями настроены (например, Product → Category, Brand, Supplier).

Все эндпоинты протестированы, сервер запускается без ошибок.
